# タグ機能一時停止・説明文コメントアウト記録（2026-04-24）

## 目的
- タグ機能を一旦停止し、関連UI/検索条件/表示データを無効化する。
- ホームページやダッシュボード上の説明文を一時的に非表示化する。

## 実施内容（結論）
- タグ機能は**削除せずコメントアウト中心**で停止。
- 表示だけでなく、タグ関連のクエリ・送信パラメータも止めて挙動不整合を回避。
- 主要説明文をコメントアウトして文言表示を抑制。

## 変更ファイル

### 1) `frontend/src/app/(dashboard)/profile/page.tsx`
- タグ取得クエリ（`getUserTags`）をコメントアウト。
- 全タグ取得クエリ（`getTags`）をコメントアウト。
- タグ追加/削除ミューテーション（`addUserTag` / `removeUserTag`）をコメントアウト。
- プレビュー用データの `tags` を空配列固定（`tags: []`）。
- プロフィール画面の「タグ」カードUIを停止。
- ヘッダー説明文（「あなたのプロフィール情報を管理」）をコメントアウト。

### 2) `frontend/src/app/(dashboard)/home/page.tsx`
- 検索条件判定から `filters.tags` を除外。
- 検索API送信ペイロードから `tags` を除外。
- プロフィールプレビュー初期データの `tags` を空配列固定。
- ヘッダー説明文（「新しい出会いを見つけましょう」）をコメントアウト。

### 3) `frontend/src/components/features/DiscoverFilters.tsx`
- タグ関連state/ref/handlerを停止（`showTagPicker`, `tagPickerRef`, `handleTagToggle` など）。
- タグ取得クエリ（`getTags`）を停止。
- タグ選択UIブロックを停止（ピッカー、選択中バッジ表示）。

### 4) `frontend/src/app/page.tsx`（ランディングページ）
- ヒーロー説明文をコメントアウト。
- 特徴セクション説明文をコメントアウト。
- 特徴カードの「タグベースマッチング」をコメントアウト。
- そのほか安全性/CTA/フッター内の説明文をコメントアウト。

## 品質確認
- `ReadLints` にて編集対象ファイルのlint確認を実施。
- 最終状態でエラーなし。

## 補足（復帰方針）
- 今回は機能削除ではなく「一時停止」実装。
- 再開時はコメントアウト復帰で元の動線に戻せる。
- ただし本格停止を継続する場合は、未使用import/型/APIを段階的に削除して恒久化するのが望ましい。
